Application Documentation

  • This project builds the back-end of a "movies" application.
  • This application will allow you to select different movies, directors, and genres.
  • Coming soon, you will be able to sign up and create a list of your favorite movies. This way you can learn more about the movies you watch and are interested in!

Business Logic URL HTTP Method Request body data format Response body data format
Return a list of all movies /movies GET None A JSON object holding the list of movies: { id: 1, title: "Braveheart"}
Return data about a single movie /movies/[movie title] GET None A JSON object data about a single movie: { id: 1, title: "Braveheart"}
Return data about a specific genre /genre/[name] GET None A JSON object holding data about a genre: { id: 1, name: "Thriller", description: "Like horror"}
Return data about a specific director /director/[name] GET None A JSON object holding data about a specific director: { id: 1, name: "Speilberg", movies: "ET"}
Allow a new user to register /users POST A JSON object holding data about a specific user to add: { name: "coolFox1", password: "ApplePie4"} A JSON object holding data about the user added, including ID: { id: 1, name: "coolFox1", password: "ApplePie4"}
Allows users to update their info /users/[userName] PUT A JSON object holding updated information about user: {Username: CoolGuy, Password: Chill11, etc.} A JSON object holding the updated user {Username: BigGuy, Password: unlock123, etc.}
Add data for a new movie to user favorites /user/[userName]/movies/[movie id] POST None A JSON object holding data about the user with the new favorite movie {Username: RecklessG, Favorite Movies: ID123, etc.}
Remove a movie /movies/[ID] DELETE None A text message indicating the movie was removed.
Deregister a user /users/[ID] DELETE None A text message indicating whether the student was successfully removed.
